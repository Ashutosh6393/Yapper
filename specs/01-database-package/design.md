# 01 · Database Package — Design

## Goal State (acceptance)
1. `@yapper/db` exports a typed Drizzle client (`db`) and the full `schema`.
2. `bun run db:generate` produces SQL migrations; `bun run db:migrate` applies them to the Compose Postgres.
3. After migrate, tables exist: `note`, `note_doc`, `note_collaborator` (+ enums).
4. A throwaway script (or test) can `insert` a note and `select` it back, type-checked end to end.
5. `api` and `socket` can `import { db, schema } from "@yapper/db"` and type-check.

> Better Auth's own tables (`user`, `session`, `account`, `verification`) are **not** created here —
> they are generated by Better Auth's schema/CLI in slice 02. This slice references `user.id`
> via a column type but the FK is wired when the auth tables land (02). Keep `owner_id`/`user_id`
> as `text`/`uuid` columns now; add FK constraints in 02's migration.

## Scope
**In:** Drizzle setup in `packages/db`, connection client (node-postgres or Bun SQL), `drizzle.config.ts`,
schema for the three app tables + enums, migration scripts, a minimal smoke test.
**Out:** Better Auth tables (02), any query/repository helpers beyond the client (those live with their features).

## Schema (Drizzle)
```
enum note_access      = ('private','view','edit')   default 'private'
enum collab_status    = ('active','revoked')        default 'active'

note
  id          uuid pk default gen_random_uuid()
  owner_id    uuid not null            -- FK -> user.id added in slice 02
  title       text not null default 'Untitled'
  preview     text not null default ''
  access      note_access not null default 'private'
  share_token text unique              -- null when private; rotated on revoke
  created_at  timestamptz not null default now()
  updated_at  timestamptz not null default now()

note_doc                               -- heavy CRDT blob, separate table
  note_id     uuid pk references note(id) on delete cascade
  state       bytea not null           -- Y.encodeStateAsUpdate(doc)
  updated_at  timestamptz not null default now()

note_collaborator
  id          uuid pk default gen_random_uuid()
  note_id     uuid not null references note(id) on delete cascade
  user_id     uuid not null            -- FK -> user.id added in slice 02
  status      collab_status not null default 'active'
  joined_at   timestamptz not null default now()
  last_access timestamptz not null default now()
  unique(note_id, user_id)
```
Indexes: `note(owner_id)`, `note(share_token)` (unique already), `note_collaborator(user_id, status)`.

## Deliverables
```
packages/db/
  package.json            # @yapper/db; deps: drizzle-orm, pg (or bun:sql), drizzle-kit (dev)
  drizzle.config.ts       # dialect postgres, schema path, out=./drizzle
  src/client.ts           # creates + exports `db` from DATABASE_URL
  src/schema.ts           # tables + enums above, exported as `schema`
  src/index.ts            # re-export db, schema, types
  drizzle/                # generated migrations (committed)
  tsconfig.json
```

## Implementation tasks
1. `packages/db/package.json` + tsconfig (extends node base) → verify resolves in workspace.
2. `src/schema.ts` enums + 3 tables → verify `drizzle-kit generate` emits SQL.
3. `src/client.ts` reading `DATABASE_URL` → verify connects to Compose PG.
4. `drizzle.config.ts` + `db:generate`/`db:migrate` scripts → verify migrate creates tables (`\dt`).
5. Smoke test (`bun test`) insert+select a note → verify round-trip + types.

## Test plan
- `packages/db` smoke test: open client, insert note, select by id, assert fields, cleanup.
- Migration applies cleanly on a fresh Compose volume.

## Risks / notes
- Choose pg driver: prefer `node-postgres` (`pg`) for Drizzle maturity on Bun; if Bun's native SQL is
  smoother, document the switch in decisions. Pick one, keep it consistent.
- `gen_random_uuid()` needs `pgcrypto`/pg16 (built-in in PG16). Confirm extension or use `uuid` default in app.
- FK to `user` deferred to 02 — note this clearly so 02 adds the constraint migration.
