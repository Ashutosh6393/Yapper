# CLAUDE.md — 01 · Database Package

## Project Context
`@yapper/db`: the single Drizzle schema + client shared by `api` (metadata, collaborators) and
`socket` (CRDT blob persistence). This slice creates the three app-owned tables only.

## Before Starting Work
1. Read `design.md` (schema is the contract).
2. Confirm Compose Postgres from slice 00 is running (`docker compose up -d`).
3. Check `implementation.md`.

## Code Patterns
- One `schema.ts` exporting every table + enum; `client.ts` builds `db` from `DATABASE_URL`.
- Migrations are generated by `drizzle-kit`, committed under `packages/db/drizzle/`, never hand-edited.
- Export inferred types (`typeof note.$inferSelect`) for consumers; no `as any`.
- `note_doc.state` is `bytea` (binary) — keep it in its own table, never select it in list queries.

## Don't
- Don't create Better Auth tables here (that's slice 02) — but leave `owner_id`/`user_id` columns ready.
- Don't add per-feature query helpers here; only the client + schema.
- Don't hand-edit generated migration SQL.
- Don't select `note_doc.state` in dashboard/list paths.
